Branches. A git branch is could be compared to PROD, QA, and DEV. The master branch being PROD and all other branches would be considered QA and DEV. The changes made on other branches will not be part of the Master branch unless there is a git pull request.

Merge. Git merge will combine the QA or DEV branch with the Master(PROD) branch. All changes that were only on the QA or DEV branch will now show on the Master branch

Commit. A git commit is a way of creating a "snapshot" of the existing code including all the changes that were made up to the commit

Tags. A git tag is used to tag parts of the repository's history. Thus like commits, the changes made have a comment. Tags used to update the code and mark important points.

Repositories. A repository is a folder where all the changes are being stored. When starting a project, you create a repository to upload all code or files into the specific repository. Only the files that are pushed into the repository will be there.
